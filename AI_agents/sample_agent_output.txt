AGENTE:
A análise de microbiota diferencial é um dos pilares da pesquisa em microbioma, visando identificar microrganismos (ou grupos taxonômicos) cuja abundância difere significativamente entre diferentes grupos experimentais (e.g., doença vs. saúde, tratamento vs. controle). Utilizar dados do E-MTAB (EMBL-EBI Metagenomics Archive) no R para essa análise envolve várias etapas, desde a aquisição dos dados brutos até a interpretação estatística e biológica.

A seguir, detalho o procedimento completo e detalhado:

---

### **Introdução à Análise de Microbiota Diferencial com Dados E-MTAB no R**

O E-MTAB é parte do ArrayExpress e do EMBL-EBI, um repositório de dados de experimentos de ómicas, incluindo dados de sequenciamento de 16S rRNA ou metagenômica shotgun. Esses dados, frequentemente, estão em formato FASTQ (reads brutos), o que exige um pipeline bioinformático robusto antes da análise estatística no R.

**Objetivo:** Identificar ASVs (Amplicon Sequence Variants) ou OTUs (Operational Taxonomic Units) que estão diferencialmente abundantes entre grupos definidos no seu estudo.

**Pré-requisitos:**
*   Conhecimento básico de R e RStudio.
*   Familiaridade com conceitos de microbioma e sequenciamento de nova geração.
*   Capacidade de instalar e carregar pacotes no R.
*   Recursos computacionais adequados (RAM e CPU) para o processamento de sequências.

---

### **Etapa 1: Aquisição e Preparação dos Dados do E-MTAB**

#### **1.1. Identificação do Estudo no E-MTAB**
*   **Navegação:** Acesse o site do ArrayExpress (www.ebi.ac.uk/arrayexpress/) ou o EMBL-EBI Metagenomics (www.ebi.ac.uk/metagenomics/).
*   **Busca:** Procure pelo identificador E-MTAB específico ou use termos de busca relacionados ao seu interesse (e.g., "gut microbiome", "16S rRNA", "mouse colitis").
*   **Detalhes do Estudo:** Verifique cuidadosamente a descrição do experimento, o tipo de sequenciamento (16S rRNA V3-V4, V4, etc., ou metagenômica shotgun), o número de amostras, as condições experimentais e, o mais importante, os metadados associados.

#### **1.2. Download dos Dados Brutos (FASTQ)**
*   **ENA Browser:** Uma vez que você encontra o experimento no ArrayExpress, ele geralmente terá links para o European Nucleotide Archive (ENA). O ENA permite baixar os arquivos FASTQ.
*   **Ferramentas de Download:**
    *   **Manual:** Baixar via navegador, mas impraticável para muitos arquivos.
    *   **`wget` ou `curl`:** Ferramentas de linha de comando para download.
    *   **`aspera connect`:** Ferramenta rápida para grandes volumes de dados, frequentemente recomendada pelo ENA.
    *   **`sra-tools` (se os dados estiverem também no SRA):** Pode usar `fastq-dump` do pacote SRA Toolkit se o estudo for espelhado no NCBI SRA.

    **Exemplo (linha de comando para `wget`):**
    ```bash
    # Exemplo de download de um arquivo FASTQ (substitua pelo URL real do ENA)
    wget ftp://ftp.ebi.ac.uk/vol1/fastq/ERR000/ERR000000/ERR000000_1.fastq.gz
    wget ftp://ftp.ebi.ac.uk/vol1/fastq/ERR000/ERR000000/ERR000000_2.fastq.gz
    ```

#### **1.3. Organização dos Dados e Metadados**
*   **Estrutura de Pastas:** Crie uma pasta organizada para o projeto (e.g., `projeto_microbiota/dados_brutos`, `projeto_microbiota/metadados`, `projeto_microbiota/resultados`).
*   **Metadados:** Os metadados são CRUCIAIS. Eles contêm informações sobre cada amostra (e.g., ID da amostra, grupo experimental, idade, sexo, condição da doença, tratamento).
    *   Normalmente, o E-MTAB fornece um arquivo de metadados (TXT, CSV ou TSV). Baixe-o e revise-o.
    *   **Padronização:** Certifique-se de que o ID da amostra nos metadados corresponde exatamente aos nomes dos seus arquivos FASTQ (ou aos IDs de amostra que você extrairá deles).
    *   **Formato:** O metadado deve ser um arquivo tabular onde cada linha é uma amostra e cada coluna é uma variável descritiva.

### **Etapa 4: Análise de Abundância Diferencial**

Esta é a etapa central. Existem vários métodos, cada um com suas premissas. Os mais comuns para dados de sequenciamento de contagens são `DESeq2` e `ALDEx2`.

#### **Considerações Importantes:**
*   **Dados de Contagem:** Os dados brutos de ASVs são contagens inteiras.
*   **Composicionalidade:** A natureza composicional dos dados de microbioma (frações, soma 1) pode ser um desafio. Alguns métodos lidam com isso intrinsecamente (`ALDEx2`), enquanto outros (como `DESeq2`) foram desenvolvidos para RNA-seq mas se mostraram robustos para contagens de microbiota.
*   **Inflação de Zeros:** Muitos ASVs têm contagem zero em muitas amostras.
*   **Normalização:** Essencial para comparar amostras com diferentes profundidades de sequenciamento. `DESeq2` e `edgeR` têm suas próprias normalizações (`size factors`).
*   **Correção para Múltiplos Testes:** Absolutamente crítica (`FDR` - False Discovery Rate).

#### **4.1. Método 1: `DESeq2`**
`DESeq2` modela dados de contagem usando distribuições binomial negativa e é robusto a variação na profundidade de sequenciamento através de fatores de tamanho.

```R
library(DESeq2)

# Converter o objeto phyloseq para um DESeqDataSet
# Certifique-se de que a variável de grupo experimental está no sample_data
dds <- phyloseq_to_deseq2(ps_clean, ~ GrupoExperimental) # Substitua GrupoExperimental

# Rodar o DESeq2
# Note: Se houver amostras com 0 reads totais, DESeq2 irá reclamar.
# phyloseq::prune_samples(sample_sums(ps_clean) > 0, ps_clean) resolve isso.
dds <- DESeq(dds)

# Obter resultados
# Substitua "GrupoExperimental" e os nomes dos níveis que você quer comparar
# Ex: "GrupoB" vs "GrupoA"
res <- results(dds, contrast=c("GrupoExperimental", "GrupoB", "GrupoA"), alpha=0.05)
res <- res[order(res$padj), ] # Ordenar por p-valor ajustado

# Visualizar resultados
summary(res)
head(res)

# Adicionar informações taxonômicas aos resultados
res_df <- as.data.frame(res)
res_df$ASV <- rownames(res_df)
tax_df <- as.data.frame(tax_table(ps_clean))
res_df <- merge(res_df, tax_df, by.x = "ASV", by.y = 0)

# Filtrar ASVs significativamente diferentes (FDR < 0.05 e fold change > 2)
sig_ASVs <- subset(res_df, padj < 0.05 & abs(log2FoldChange) > 1) # log2FC de 1 = fold change de 2x
head(sig_ASVs)
```

#### **4.2. Método 2: `ALDEx2`**
`ALDEx2` é projetado especificamente para dados composicionais, usando a transformação Log-Ratio de Abundância Centrada (CLR) e diferentes testes estatísticos (t-test, Wilcoxon, GLM).

```R
library(ALDEx2)

# Extrair a tabela de contagens e metadados
count_data <- as.data.frame(otu_table(ps_clean))
metadata_aldex <- as.data.frame(sample_data(ps_clean))

# Obter o vetor de condições para ALDEx2
conditions <- metadata_aldex$GrupoExperimental # Substitua GrupoExperimental

# Rodar ALDEx2 (exemplo com t-test)
# Para dados com mais de 2 grupos, considere aldex.glm
x <- aldex.clr(count_data, conditions, mc.samples=128, denom="all", verbose=TRUE)
x_ttest <- aldex.ttest(x, paired.test=FALSE, hist.plot=TRUE, verbose=TRUE)
x_kw <- aldex.kw(x) # Para mais de 2 grupos (Kruskal-Wallis)

# Visualizar resultados
head(x_ttest)

# Adicionar informações taxonômicas
x_ttest_df <- as.data.frame(x_ttest)
x_ttest_df$ASV <- rownames(x_ttest_df)
tax_df <- as.data.frame(tax_table(ps_clean))
x_ttest_df <- merge(x_ttest_df, tax_df, by.x = "ASV", by.y = 0)

# Filtrar ASVs significativamente diferentes (FDR < 0.05 e efeito direcional (ex: we.ep < 0.05))
# we.eBH é o FDR ajustado para o efeito t-test (Welch)
sig_ALDEx_ASVs <- subset(x_ttest_df, we.eBH < 0.05 & abs(diff.btw) > 0) # diff.btw é a diferença das médias CLR
head(sig_ALDEx_ASVs)
```

**Escolha do Método:**
*   **`DESeq2` e `edgeR`:** Geralmente são a primeira escolha para dados de contagem se as premissas forem razoavelmente atendidas. Eles são poderosos e testados.
*   **`ALDEx2`:** Excelente para dados composicionais e quando você quer ser mais rigoroso com a natureza relativa das abundâncias. Útil quando `DESeq2` tem dificuldades ou se a composição é uma preocupação primária.
*   Outros métodos como `ANCOM-BC` são também opções válidas, especialmente desenhados para dados composicionais de microbioma.
